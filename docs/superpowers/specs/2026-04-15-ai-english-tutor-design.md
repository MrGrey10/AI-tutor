# AI English Tutor — Design Spec
**Date:** 2026-04-15

## Overview

A Next.js web app that acts as an AI English conversation tutor for intermediate learners. The user speaks via their microphone; the browser transcribes their speech to text, sends it to the AI tutor, and reads the tutor's response aloud. The tutor corrects grammar and vocabulary mistakes inline during conversation and provides a summary of all mistakes at the end of each session.

Deployed on Netlify. Each session is standalone — no persistence between sessions.

---

## Target User

Intermediate English learners who want to practice real conversation. Not beginners, not advanced — someone who can hold a conversation but needs corrections and guided practice.

---

## Architecture

```
Browser
  ├── Web Speech API (SpeechRecognition) → transcribes voice to text
  ├── Web Speech API (SpeechSynthesis)   → reads tutor response aloud
  └── React UI (Next.js)
        └── fetch POST /api/chat
              └── Next.js API Route (Netlify Function)
                    └── Groq SDK → llama-3.3-70b-versatile
```

- The Groq API key lives only on the server (Netlify Function). It is never exposed to the browser.
- Full conversation history is held in React state (array of `{ role, content }` objects) and sent with each request.
- No database, no authentication, no server-side state.

## UI & User Flow

### Session Start Screen
- Welcome message
- Two options: **Free chat** or **Choose a topic**
- Topic list (preset): Job interview, Travel, Daily life, Technology, Hobbies
- Selecting a topic seeds the first tutor message to set context for the conversation

### Main Conversation Screen
- Chat bubble layout — user messages on the right, tutor messages on the left
- Tutor correction notes appear as a distinct bubble style (different background color, prefixed with "By the way —")
- Mic button at the bottom:
  - **Desktop:** click to start recording, click again to stop and send (toggle)
  - **Mobile:** tap and hold to record, release to send (push-to-talk)
- **End session** button opens the summary screen

### Summary Screen
- Full list of mistakes made during the session paired with corrected versions
- Generated by sending a final summary-request message to the model (which has full conversation history)
- **Start new session** button resets all state

### Browser Compatibility Warning
- If `window.SpeechRecognition` (and `window.webkitSpeechRecognition`) are not available, show a clear banner: "Your browser doesn't support voice input. Please use Chrome or Edge."

## Tutor Behavior (System Prompt)

The API route sends a fixed system prompt with every request:

> You are a friendly, encouraging English conversation tutor for intermediate learners. Respond naturally as a conversation partner — keep replies concise (2–4 sentences). If the user makes a grammar or vocabulary mistake, first respond to what they said naturally, then add a short correction: "By the way — you could say: [corrected version]." Never refuse to engage with a topic; always keep the conversation going. If a topic was selected at the start, keep responses relevant to that topic for the first few exchanges, then let the conversation flow naturally.

## API Route: `/api/chat`

**Request body:**
```json
{
  "messages": [{ "role": "user" | "assistant", "content": "string" }],
  "userText": "string"
}
```

**Behavior:**
1. Prepend the system prompt to the messages array
2. Append the new user message
3. Call Groq (`llama-3.3-70b-versatile`, `stream: false`)
4. Return `{ "reply": "string" }`

**Summary trigger:**
When the user ends the session, the client appends a special user message: `"Please summarize all the grammar and vocabulary mistakes I made in this session with their corrections."` — sent through the same `/api/chat` endpoint.

**Environment variable:**
- `GROQ_API_KEY` — set in Netlify dashboard, never in client code

---

## Deployment

- **Framework:** Next.js (App Router)
- **Host:** Netlify — auto-detects Next.js, runs API routes as Netlify Functions
- **Config:** No special `next.config.js` needed for basic Netlify deploy
- **Env vars:** `GROQ_API_KEY` set in Netlify environment settings

---

## Out of Scope

- User accounts / authentication
- Saving or replaying past sessions
- Custom voice selection for TTS (uses browser default)
- Mobile app (web only, but mobile-responsive)
- Whisper or any server-side STT (browser Web Speech API only)
